### Four Deviations — All ACCEPTED

**Deviation 1: MSW interceptor.apply() at module load time, not in beforeAll**

ACCEPT. The OpenAI SDK constructs fetch references eagerly at describe-scope. If apply() is deferred to beforeAll, the SDK clients are created before the patch lands. The design explicitly accepts setupFiles as the activation mechanism. Test isolation is sound via afterEach/afterAll cleanup.

**Deviation 2: 401 instead of 500 for error tests**

ACCEPT. The spec explicitly specifies "interceptor returns 401" in scenario text. 401 causes immediate AuthenticationError (no retry), satisfying raw-passthrough identically to 500 without test timeouts. The binding contract is "raw propagation," not the specific status code.

**Deviation 3: Positional `vitest run tests/integration` instead of `--dir tests/integration`**

ACCEPT. With an explicit `include` pattern in vitest.config.ts, the `--dir` flag is ignored by Vitest. The positional form correctly overrides include at the CLI level. Tasks.md was based on incorrect assumptions about Vitest's `--dir` behavior. The outcome (exit 0, all integration tests skipped) satisfies REQ-TEST-INTG-001.

**Deviation 4: Azure models.list synchronous throw on Promise return type**

ACCEPT. TypeScript permits synchronous throw on `Promise<ModelInfo[]>` return type. The implementation throws before returning — the throw propagates to the catch block correctly. Tests use sync `toThrow`. The spec explicitly permits "throw synchronously or reject." Behavior is correct.

## Known Limitations Carried Forward

**Azure test() Permanently False (Phase 2):**
Since `models.list()` always throws on Azure and `test()` is implemented as `try { await models.list(); return true } catch { return false }`, Azure's `test()` will always return `false` in this change. This is a documented known limitation, not a defect. Future `helix-azure-config-v2` will enable meaningful Azure `test()` once ARM credentials path is available.

**Errors Propagate Raw (Phase 2):**
All SDK errors (OpenAI APIError, Azure errors, custom endpoint errors) propagate unchanged. `HelixError` structured error model is deferred to `helix-error-model`, a separate change. Callers catch raw `openai` SDK error types until that change lands.

## Post-Archive Correction (in-flight before commit)

**Date**: 2026-04-28
**Author**: post-archive pre-commit refactor

### Rationale

After archiving but before committing, the decision was made to eliminate MSW (`@mswjs/interceptors`) from the unit-test suite and replace it with `vi.mock("openai")`. The core insight: the three adapters (`openai`, `azure`, `custom`) are thin pass-throughs to the OpenAI SDK. HTTP-level interception tests the network stack, not the adapters. Module-mocking the SDK at the import boundary is more honest — it directly controls what the adapter receives and asserts what it forwards, without any HTTP round-trip illusion.

MSW was overkill for this use case. It added a devDependency, a global setup file, and a `BatchInterceptor` lifecycle that had to be applied at module-load time (see Deviation 1 in the archive report) to avoid construction-order bugs. `vi.mock` removes all of that.

### Code Changes

- **DELETED**: `src/internal/providers/__tests__/_msw.ts` — BatchInterceptor setup and `respondWith` helper. No longer needed.
- **REWRITTEN**: `src/internal/providers/__tests__/openai.test.ts` — 13 tests preserved, same assertions, MSW replaced with `vi.mock("openai")` + `vi.fn()` method mocks (`mockResponsesCreate`, `mockFilesCreate`, `mockFilesList`, `mockFilesDelete`, `mockModelsList`). Constructor call now assertable via `expect(OpenAI).toHaveBeenCalledWith(...)`.
- **REWRITTEN**: `src/internal/providers/__tests__/azure.test.ts` — 10 tests preserved, same assertions including exact ARM message string (REQ-AZ-004). Uses `vi.mock("openai")` + `AzureOpenAI` mock. `models.list` synchronous throw tests work unchanged since the adapter throws before touching the SDK.
- **REWRITTEN**: `src/internal/providers/__tests__/custom.test.ts` — 11 tests preserved, same assertions including exact REQ-CUSTOM-004 error messages for files.* stubs. Uses `vi.mock("openai")` + `OpenAI` mock. Added `baseURL` assertion via `expect(OpenAI).toHaveBeenCalledWith(expect.objectContaining({ baseURL: ... }))`.
- **MODIFIED**: `vitest.config.ts` — removed `setupFiles` entry pointing to `_msw.ts`. No setup file needed.
- **MODIFIED**: `package.json` — removed `@mswjs/interceptors` from `devDependencies`.
- **MODIFIED**: `package-lock.json` — regenerated via `npm uninstall @mswjs/interceptors`; 7 packages removed.

No adapter source files were touched (`openai.ts`, `azure.ts`, `custom.ts`, `vertex.ts` are bit-identical). No integration tests were touched. No public surface was modified.

### Spec Changes

Live specs in `openspec/specs/` were corrected to reflect the new mocking approach. The delta specs inside this archive folder (`openspec/changes/archive/helix-providers-phase-2/specs/`) are left as historical record — they describe the state at archive time and are not retroactively modified.
